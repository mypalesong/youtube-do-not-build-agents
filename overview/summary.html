핵심 요약 ​

이 문서는 Barry Zhang과 Mahesh Murag의 발표에서 제시된 8가지 핵심 논점을 요약한 것이다. 각 항목은 발표에서 직접 등장한 발언과 맥락을 바탕으로 정리되었다.

1. 에이전트는 똑똑하지만, 전문성은 없다 ​

발표의 출발점은 단순한 관찰이다. 오늘날의 에이전트는 놀라운 지능을 갖췄지만, 실무가 요구하는 전문성은 부족하다. Zhang과 Murag은 이를 "세무 코드를 처리할 때, 300 IQ 수학 천재가 아니라 경력 20년의 세무 전문가를 고르겠다"는 비유로 설명한다. 문제는 모델이 덜 똑똑해서가 아니라, 도메인 맥락이 사전에 주어지지 않기 때문이다.

2. 스킬 = 폴더. 그게 전부다 ​

스킬의 정의는 놀랄 만큼 단순하다. "절차적 지식을 패키지로 묶은 파일 모음. 쉽게 말하면 폴더다." SKILL.md라는 마크다운 파일 하나만 있으면 최소한의 스킬이 된다. Git으로 관리하고, Google Drive로 공유하고, ZIP으로 배포할 수 있다. 별도의 SDK도, 전용 IDE도, 새로운 런타임도 필요 없다. 이 단순함은 실수가 아니라 의도된 설계다. 누구나 만들 수 있어야 하기 때문이다.

3. 도구(Tool) 말고 스크립트 -- 수정 가능성과 자기 문서화 ​

기존의 에이전트 도구(tool)는 모호한 인터페이스와 수정 불가능한 블랙박스라는 두 가지 한계가 있었다. 스킬은 이를 스크립트로 해결한다. 코드는 자기 자신을 설명하고(self-documenting), 동작이 잘못되면 수정할 수 있다. Claude가 반복적으로 같은 Python 스크립트를 작성하던 슬라이드 스타일링 작업은 스킬로 저장된 뒤부터 일관된 실행이 보장되었다.

4. 코드는 디지털 세계의 범용 인터페이스 ​

Zhang과 Murag은 Claude Code를 만들면서 결정적인 발견을 했다. "코드는 단순한 사용 사례가 아니라, 디지털 세계와 소통하는 보편 인터페이스다." 파이낸셜 리포트 생성, 데이터 분석, 파일 정리 -- 이 모든 것이 결국 코드라는 공통 언어로 수렴한다. 그래서 Claude Code는 "코딩 에이전트"가 아니라 실은 범용 에이전트다.

5. 점진적 공개 -- 수천 개의 스킬을 담는 법 ​

수백, 수천 개의 스킬을 에이전트에 장착하면 컨텍스트 윈도우가 폭발하지 않을까? 해답은 **점진적 공개(Progressive Disclosure)**다. 시스템 프롬프트에는 스킬의 메타데이터(이름과 설명)만 노출되고, 에이전트가 필요하다고 판단할 때 비로소 SKILL.md 본문이 로드되며, 그 이후에는 스킬 안에 포함된 스크립트나 참조 파일이 필요할 때에만 읽혀 들어간다. 3단계로 나누어 컨텍스트를 얇게 유지하면서도 지식의 깊이를 확보한다.

6. 5주 만에 수천 개 -- 이미 생태계가 생겼다 ​

Anthropic이 Skills를 공개한 지 5주 만에 이미 세 갈래의 생태계가 형성되었다.

  • 기초 스킬(Foundational): Anthropic의 문서 작성 스킬, Cadence의 생명과학 연구 스킬 등
  • 서드파티 스킬(Third-party): Browserbase의 Stagehand, Notion의 워크스페이스 스킬 등
  • 엔터프라이즈 스킬(Enterprise): Fortune 100 기업들이 내부 코딩 컨벤션과 운영 플레이북을 스킬로 인코딩

주목할 점은 비개발자의 참여다. 법무, 재무, 회계, 리크루팅 담당자들이 코드 없이 스킬을 만들고 있다. 스킬은 범용 에이전트를 자신의 업무에 맞게 확장하는 민주화된 수단이 되었다.

7. 에이전트 아키텍처는 수렴하고 있다 ​

Zhang과 Murag은 2025년을 관통하며 본 현실을 이렇게 정리한다. 서로 다른 도메인 에이전트가 난립할 줄 알았지만, 아래 구조에서 모두 수렴하고 있다.

  • 에이전트 루프 -- 컨텍스트와 토큰 흐름 관리
  • 런타임 환경 -- 파일 시스템과 코드 실행
  • MCP 서버 -- 외부 세계와의 연결 (도구, 데이터)
  • 스킬 라이브러리 -- 런타임에 동적으로 로드되는 도메인 전문성

MCP가 세계와의 연결이라면, 스킬은 전문성의 담지체다. 둘은 경쟁하지 않고 보완한다. 올바른 MCP 서버와 큐레이션된 스킬 라이브러리만 있으면 새로운 수직 시장을 빠르게 개척할 수 있고, 실제로 Anthropic은 이 조합으로 금융 서비스와 생명과학 솔루션을 출시했다.

8. 스킬은 집단 지식의 컨테이너다 -- 그리고 OS의 애플리케이션 층이다 ​

가장 야심찬 비전은 마지막에 등장한다. 스킬은 한 팀, 한 조직의 암묵지를 물질화된 형태로 저장할 수 있게 한다. 새 팀원이 합류했을 때 에이전트는 이미 "이 팀이 무엇을 중시하는지, 일상 업무가 어떻게 돌아가는지"를 알고 있다. 그리고 Claude 자신도 자기 자신을 위한 스킬을 만들 수 있다(skill creator skill). Claude가 기록해 둔 것은 미래의 자기 자신이 효율적으로 사용할 수 있다.

이 구조를 컴퓨터의 역사에 비유하자면:

  • 프로세서 = 모델 (막대한 투자, 단독으로는 가치 제한)
  • 운영체제 = 에이전트 런타임 (리소스 조율, 토큰과 데이터 흐름)
  • 애플리케이션 = 스킬 (도메인 전문성과 창의적 문제 해결이 사는 곳)

1970년대 이후 컴퓨터 혁명은 운영체제가 아니라 그 위에서 피어난 수백만 개의 애플리케이션에서 이루어졌다. Zhang과 Murag이 전하는 메시지는 분명하다. 에이전트의 애플리케이션 계층이 이제 열렸다. 에이전트를 다시 만들지 말고, 스킬을 만들기 시작할 때다.
