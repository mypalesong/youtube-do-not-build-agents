주제별 분석 ​

이 발표에서 다루어진 핵심 주제들을 7개의 독립된 글로 분석했다. 각 글은 발표 원문의 인용과 함께 맥락과 의미를 한국어로 풀어낸다.

주제 목록 ​

전문성의 공백 ​

에이전트는 똑똑해졌지만 여전히 실무에서 부족하다. 왜 그런가? Zhang과 Murag은 세무 전문가 vs 수학 천재의 비유로 이 공백을 설명한다. 지능과 전문성의 차이, 그리고 왜 더 큰 모델이 해답이 아닌지를 분석한다.

스킬이란 무엇인가 ​

"폴더다." 이 놀랍도록 단순한 정의 뒤에 숨은 설계 철학. SKILL.md 파일 하나에서 시작해 수백 개 스크립트와 바이너리로 확장되는 구조, 그리고 왜 이 단순함이 민주화의 열쇠가 되는지를 해부한다.

코드, 세계와의 인터페이스 ​

Claude Code를 만들면서 얻은 핵심 통찰. 왜 코드는 단지 프로그래머의 도구가 아니라, 에이전트가 디지털 세계와 소통하는 보편 언어가 되는가? 도구(tool) 대신 스크립트를 택한 이유도 함께 다룬다.

점진적 공개 ​

수천 개의 스킬을 에이전트에 장착하면서도 컨텍스트를 지키는 설계 원리. 메타데이터 → SKILL.md → 참조 파일로 이어지는 3단계 공개가 어떻게 확장성과 효율성을 동시에 확보하는지 분석한다.

스킬 생태계 ​

5주 만에 수천 개. 기초 스킬, 서드파티 스킬, 엔터프라이즈 스킬이라는 세 갈래의 지형도. 비개발자 참여, 복잡도의 진화, 그리고 이 생태계가 보여주는 소프트웨어 산업의 새로운 층위를 조망한다.

수렴하는 에이전트 아키텍처 ​

"모든 에이전트는 결국 같은 구조로 수렴한다." 에이전트 루프 + 런타임 + MCP + 스킬이라는 4개 축. MCP와 Skills의 역할 분담, 수직 시장을 빠르게 여는 조합 전략까지 다룬다.

지속 학습과 집단 지식 ​

스킬은 에이전트를 가르치는 방법이자, 조직의 암묵지를 물질화하는 수단이다. 30일 차 Claude가 1일 차보다 나은 이유, 스킬 크리에이터 스킬, 그리고 "모델 = CPU, 런타임 = OS, 스킬 = 애플리케이션"이라는 역사적 비유를 분석한다.
